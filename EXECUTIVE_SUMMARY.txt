     • Result: User sees other users' tasks

     • All tasks from backend are blindly added to UI collection
     • If backend returns all tasks, that's what user sees

     • Missing check: if (apiTask.UserId != _userId) skip
   
   Security Impact:
     • Client accepts any task from backend without validation
     • No defensive programming
     • No safety net if backend fails to filter


4. API ENDPOINT DOCUMENTATION
   Status: ✅ Well Documented | ❌ Not Implemented in Backend
